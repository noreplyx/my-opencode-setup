# Plan Protocol Skill

When faced with a complex software engineering task, follow this protocol to ensure a well-thought-out implementation.

## Planning Requirements

For any non-trivial task, you must:

1. **Generate Multiple Options**: Create at least two distinct implementation plans, or more if the problem complexity warrants it.
2. **Summarize Each Plan**: Provide a concise summary of the approach for each plan.
3. **Analyze Trade-offs**: Detail the pros and cons (trade-offs) for each proposed plan.
4. **Recommend a Preferred Choice**: Evaluate the options and clearly state which plan you prefer and why.
5. **Identify Information Gaps**: Ask the user for any additional information or clarifications needed to refine the plan or proceed with implementation.

## Plan Structure

Each plan should include:
- **Goal**: What this specific plan intends to achieve.
- **Steps**: A detailed, sequential list of technical actions.
- **Verification**: How the implementation will be verified (e.g., specific tests, linting).
- **Trade-offs**: 
    - **Pros**: Advantages of this approach.
    - **Cons**: Disadvantages or risks.
